### **PRD: Phase-2 웹사이트 완성도 향상 및 기능 구현**

**1. 프로젝트 목표**

*   현재 개발된 웹사이트의 전반적인 완성도를 높여, 방문자에게 신뢰감 있고 '잘 만들어진' 인상을 제공한다.
*   이를 위해 **시각적 디자인 개선**과 **핵심 기능 구현**이라는 두 가지 축을 중심으로 작업을 진행한다.

**2. 주요 기능 및 요구사항**

**A. 공통 디자인 개선 (10개 페이지 대상)**

*   **페이지 배너 컴포넌트 개발:**
    *   각 서브 페이지의 주제를 나타내는 배경 이미지와 제목 텍스트로 구성된 공용 '히어로 배너' 컴포넌트를 구현한다.
    *   이를 통해 페이지의 시각적 매력과 정보 전달력을 높인다.
*   **콘텐츠 애니메이션 적용:**
    *   페이지 로드 및 스크롤 시 콘텐츠가 부드럽게 나타나는 '페이드인(Fade-in)' 효과를 적용하여 세련된 사용자 경험을 제공한다.
*   **적용 대상 페이지:**
    *   환영합니다 (4개): 인사말, 예배시간 안내, 오시는 길, 새가족 등록
    *   교회소개 (2개): 교회 비전, 섬기는 사람들
    *   교회학교 (4개): 유아부, 아동부, 중고등부, 청년부

**B. Headless WordPress 연동 기능**

*   **공통 워크플로우:** 관리자가 WordPress 대시보드에서 글/미디어를 업로드하고 지정된 카테고리를 선택하면, 웹사이트의 해당 페이지에 자동으로 콘텐츠가 게시되도록 한다.
*   **대상 페이지:**
    *   **주보 (BulletinPage):** '주보 게시판' 카테고리의 게시물을 가져와 목록으로 표시한다.
    *   **교회 소식 (ChurchNewsPage):** '교회 소식' 카테고리의 게시물을 가져와 목록으로 표시한다.
    *   **앨범 (AlbumPage):** '앨범 게시판' 카테고리의 게시물을 가져오되, 텍스트보다 **사진이 중심이 되는 갤러리 레이아웃**으로 구현한다.

**C. YouTube 연동 기능 (말씀/찬양 페이지)**

*   **대상 페이지:** 주일 설교, 찬양대, 특별 찬양
*   **데이터 소스:** 교회 YouTube 채널의 3개 재생목록 (각 페이지별 1개)
*   **핵심 기술 요구사항 (API 요청 최적화):**
    *   YouTube API의 일일 요청 횟수 제한을 준수하기 위해, **백엔드 서버에서 하루에 한 번 정해진 시간에** 3개의 재생목록 정보를 미리 가져와 **자체적으로 저장(캐싱)**한다.
    *   프론트엔드(사용자 브라우저)는 YouTube에 직접 요청하는 것이 아니라, 우리 백엔드 서버에 저장된 최신 영상 목록 데이터를 요청하여 화면에 표시한다.
*   **디자인 제안:**
    *   다른 페이지와의 통일성을 고려하여 **카드(Card) 형태의 비디오 목록 UI**를 제안한다. 각 카드에는 영상 썸네일, 제목, 재생 시간 등의 정보가 포함된다.
